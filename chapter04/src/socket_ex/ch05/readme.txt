[ 서버측 실행 흐름 ]

Vector를 이용해서 사용자에 대한 정보 저장
Vector를 이용해서 룸에 대한 정보 저장

1. 서버 실행 / 유저 입장
	포트 번호를 입력하여 서버 실행을 하면 서버가 실행된다.
	실행되고 나면 사용자의 접속을 기다리며, 사용자가 포트번호를 찾아 들어오면 연결이 된다.
	연결이 완료 되면, 유저의 정보를 들고오는 스레드가 실행됨.
	유저의 네트워크에 스트림을 통해서 소켓을 연결한다.
	처음 유저가 들어오게 되면, id를 읽어와서 입장했다고 서버측에 알림
	알리고 난후 벡터에 유저를 추가한다.
	사용자 측의 접속자 명단과 룸 명단을 업데이트 해준다.


[ 서버측 기능 ]

포트를 통해 서버 실행

방 만들기
	현재 만들어진 방과 이름이 같은지 확인하고 방을 만들어줌.
	
방 지우기
	방에 사람이 아무도 없으면 방을 지운다.

채팅에 참여하면 신규 유저의 입장 알리기

메세지 전송
	방에 있는 사람에게 메세지 전송
	원하는 유저에게만 귓속말 전송
	


[ 클라이언트 실행 흐름 ]

서버에 연결 / 입장
	포트번호와 IP주소를 입력하고 ID를 만들어 서버에 연결.
	서버에 연결이 되면 서버측으로 ID를 출력한다.
	유저 벡터에 사용자의 ID를 저장하고 리스트 화면에 추가시킴.
	





[ 클라이언트 기능 ]

서버와 연결 ( 로그인 개념 )
	포트번호와 IP주소를 가지고 서버와 연결을 한다.

메세지 기능
	참여한 방에 메세지 전송
	귓속말 기능
	
방 만들기
	방은 하나만 만들 수 있다. 방을 만들고 나면 방만들기버튼이 비활성화된다.
	이름이 같은지 체크
	
방 나가기
	현재 방에 남아있는 사람들에게 퇴장을 알려준다.
	현재 방에 남아있는 사람이 없으면 강제로 방을 없앤다.
